# Plain-English Guide

CoinFox should be understandable to a beginner without becoming vague for an
experienced contributor.

## Writing Rules

- Say what the system sees.
- Say why it matters.
- Say what would weaken the idea.
- Say when sources are stale, missing, or mixed.
- Avoid fake certainty.
- Avoid exchange-action language unless a command actually places an order.

## Good Pattern

```text
BTC is leaning long because price is holding above recent support and sentiment
is improving. The thesis weakens below 67250 because that would break the recent
support structure.
```

## Weak Pattern

```text
BTC is bullish. Enter now.
```

That is too certain and does not explain the evidence.

## Required Readout Pieces

Every major readout should make these easy to find:

- bias;
- confidence;
- top drivers;
- thesis check;
- source health;
- timestamp.

## Glossary

Use [terms.md](terms.md) for shared definitions. If a user-facing term is not in
the glossary, either add it or avoid it.
