# 🦊 Hall of Foxes — coinfox Contributors

Every fox here made coinfox sharper. **Adding your name takes 5 minutes:**

```bash
python -m coinfox new-source my-source --author your_github_handle
```

That scaffolds a working module with your name pre-baked in `CONTRIBUTORS`,
auto-registers it in the intel aggregator, and your handle shows up in
`python -m coinfox credits` and in the footer of every `coinfox intel` run.

## Source authors

Every source module under `src/coinfox/sources/` declares its own
`CONTRIBUTORS` list. That list is the source of truth — this table is
generated by `python -m coinfox credits`. PR your name in!

## Want to be listed?

Pick one:

- 🔌 **Add a source** — see `python -m coinfox new-source --help` or
  [CONTRIBUTING.md](CONTRIBUTING.md). Live BTC data sources are highest impact.
- 🧠 **Improve the model** — `src/coinfox/model.py` welcomes better weighting,
  new indicators, or smarter aggregation. Add yourself to the file's
  `CONTRIBUTORS` list.
- 🎨 **Polish the dashboard** — Rich panels, color tweaks, new views.
- 📚 **Docs / examples / screenshots** — also counts!

All contribution roles (`source`, `indicator`, `model`, `ui`, `docs`,
`founder`) are recognized equally in the credits view.

## Attribution format

Every module that you author or substantially improve should carry a
`CONTRIBUTORS` block at the top:

```python
CONTRIBUTORS = [
    {"name": "Your Name", "github": "yourhandle", "role": "source",
     "added": "2026-06",
     "contribution": "short description of what this brings to coinfox"},
]
```

When you improve someone else's module, **append** to the list rather than
replacing. Credit is additive — the fox remembers everyone.
